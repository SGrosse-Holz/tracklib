1""" 

2Bayesian MSD fitting 

3 

4Any *valid* MSD function MSD(Δt) defines a stationary Gaussian process with the 

5appropriate correlation structure. Therefore, instead of fitting the graph of 

6"empirically" calculated MSDs, we can perform Bayesian inference of parametric 

7MSD curves, using the Gaussian process likelihood function. Specifically, we 

8discriminate two cases, depending on what exactly is stationary: 

9 

10 + the trajectories themselves might be sampled from a stationary process (e.g. 

11 distance of two points on a polymer). In terms of the MSD, the decisive 

12 criterion is ``MSD(inf) < inf``. In this case, it is straightforward to 

13 prove the following relation between the MSD μ and autocovariance γ of the 

14 process: 

15 

16 .. code-block:: text 

17 

18 μ(k) = 2*( γ(0) - γ(k) ) 

19 

20 Thus, the full autocovariance function can be obtained from the MSD and the 

21 steady state covariance ``γ(0)``. For decaying correlations (``γ(k) --> 0 as 

22 k --> ∞``) we furthermore see that ``2*γ(0) = μ(∞)`` is the asymptotic value 

23 of the MSD. Finally, this allows us to calculate the covariance matrix of 

24 the process as 

25 

26 .. code-block:: text 

27 

28 C_ij := <x_i*x_j> 

29 = γ(|i-j|) 

30 = γ(0) - 1/2*μ(|i-j|) 

31 

32 We call this case a steady state of order 0. 

33 + in many cases (e.g. sampling a diffusing particle's position) the 

34 trajectories itself will not be stationary, but the increment process is. In 

35 this case the Gaussian process of interest is the one generating the 

36 increments of the trajectory, whose autocorrelation is the second derivative 

37 of the MSD: 

38 

39 .. code-block:: text 

40 

41 γ(k) = 1/2 * (d/dk)^2 μ(k) 

42 

43 where derivatives should be understood in a weak ("distributional") sense. 

44 More straightforwardly, the correlation matrix of the increments is given by 

45 

46 .. code-block:: text 

47 

48 C_ij := <(x_{i+1} - x_i)(x_{j+1}-x_j)> 

49 = 1/2 * ( μ(t_{i+1} - t_j) + μ(t_i - t_{j+1}) 

50 -μ(t_{i+1} - t_{j+1}) - μ(t_i - t_j) ) 

51 

52 where by definition we let ``μ(-k) = μ(k)``. In this case, we talk about a 

53 steady state of order 1. 

54 

55In any case, the covariance matrix ``C`` (potentially together with a mean / 

56drift term for steady state order 0 / 1 respectively) defines a Gaussian 

57process, which lets us assign a likelihood to the generating MSD. Via this 

58construction, we can perform rigorous Bayesian analysis, cast in the familiar 

59language of MSDs. 

60 

61This module provides a base class for performing such inferences / fits, namely 

62`Fit`. We also provide a few example implementations of fitting schemes in the 

63`lib` submodule. Finally, the `Profiler` allows to explore the posterior once a 

64point estimate has been found, by tracing out either conditional posterior or 

65profile posterior curves in each parameter direction. Note that you can also 

66just sample the posterior by MCMC. 

67

77... import numpy as np 

78... from scipy import stats

80... class PosteriorSampler(tl.util.mcmc.Sampler): 

81... def __init__(self, profiler, mci=None, stepsize=0.1): 

82... \"\"\" 

83... `mci` is the output of a previous run of the profiler (if there was 

84... one). This is used to set the stepsize of the MCMC according to the 

85... guesses on the posterior shape from those results. 

86... \"\"\"

88...  

89... fix_values = self.profiler.fit.fix_values 

90... self.min_target_from_fit = self.profiler.fit.get_min_target(fix_values=fix_values) 

91...  

92... # Figure out, which parameters are actually independent 

93... ifix = [i for i, _ in fix_values] 

94... i_independent = np.array([i for i in range(len(self.profiler.fit.bounds)) if i not in ifix]) 

95...  

96... # Set stepsizes (if not given by mci) 

97... self.stepsizes = np.array(len(i_independent)*[stepsize]) 

98... if mci is not None: 

99... for iparam in range(len(mci)): 

100... if not np.any(np.isnan(mci[iparam])) and iparam in i_independent: 

101... m, *ci = mci[iparam] 

102... step = np.min(np.abs(ci - m)) 

103... self.stepsizes[i_independent == iparam] = step 

104...  

105... def get_initial_values(self): 

106... p0 = self.profiler.point_estimate['params'].copy() 

107... ifix = np.array([i for i, _ in self.profiler.fit.fix_values]) 

108... p0[ifix] = np.nan 

109... return p0[~np.isnan(p0)] 

110...  

111... # Implement MCMC interface 

112... def logL(self, params): 

113... return -self.min_target_from_fit(params) 

114...  

115... def propose_update(self, current_params): 

116... i_update = np.random.choice(len(current_params)) 

117... new_params = current_params.copy() 

118... step_dist = stats.norm(scale=self.stepsizes[i_update]) 

119... step = step_dist.rvs() 

120... new_params[i_update] += step 

121...  

122... return new_params, 1, 1 # don't have to care about fwd/bwd probabilities, bc Gaussian is symmetric
